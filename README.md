# Тестовое задание: сервис скачивания и анализа файлов

Перед вами API, которое раздаёт каталог текстовых файлов. Ваша задача — написать собственный сервис с простым веб-интерфейсом, который скачает весь каталог через это API и посчитает статистику по содержимому файлов.

**Документация API:** [Swagger UI (`/docs`)](https://www.google.com/search?q=docs) · [`openapi.json`]

## Как устроено API

* `GET /api/files/names` — возвращает от 3 до 9 случайных имён файлов, которые вы ещё не отметили как скачанные. Пустой список означает, что вы скачали всё.
* `POST /api/files/download` — скачивание файлов по именам, не более 3 за один запрос, ответ — ZIP-архив.
* `POST /api/files/downloaded` — отметка «эти файлы я скачал». Только после неё файлы исчезают из выдачи имён.

По умолчанию вы идентифицируетесь по IP-адресу; при желании передавайте собственный идентификатор в заголовке `X-Candidate-Id`.

> **Ограничения:** Сервис намеренно не сообщает, сколько всего файлов в каталоге, и не даёт скачать всё разом. Частота запросов ограничена: при превышении лимита вы получите `429`, а особо настойчивые будут заблокированы на 30 минут (ответ `403` с временем до разблокировки). Учитывайте это в своём клиенте — понадобится корректная обработка ошибок и пауз (смотрите заголовок `Retry-After`).


## Что нужно сделать

Создайте сервис с простым UI из двух частей.

### 1. Страница скачивания

На любой странице сервиса разместите кнопку **«Скачать данные»**, которая запускает процесс скачивания документов через API.

* Процесс продолжается до тех пор, пока ручка имён не вернёт пустой ответ — вам нужно самостоятельно понять, когда каталог скачан полностью.
* Приветствуется визуализация показателей:
* время старта процесса по Новосибирску (НСК);
* «получено N названий файлов, скачиваю / скачано M из N».



### 2. Страница скачанных файлов и расчётов

* Списком отобразите скачанные файлы: имя файла и время его скачивания.
* Сортировка по времени скачивания.
* Пагинация.
* Выбор файлов: несколько точечно, все на странице, вообще все.
* Кнопка **«Произвести расчёты»** для выбранных файлов.

## Расчёты

Содержимое файлов однотипно: одна строка из 500 символов-цифр от 0 до 9 (пример фрагмента: `2395778969177858327525074678250561522341…`). По нажатию кнопки ваш сервис должен распарсить выбранные файлы и показать:

* **общую статистику** — сколько раз встретилась каждая цифра во всех выбранных файлах;
* **статистику по файлам** — сколько раз встретилась каждая цифра в каждом файле.
